# 증명서발급 페이지(certPrint) 개발 계획

## 개요
grdList 페이지를 참조하여 증명서발급 페이지를 신규로 추가합니다.

## 파일 구조 설계
- [ ] public_html/certPrint.php (메인 페이지)
  - 참조: grdList.php (모달 팝업, 검색 영역, 테이블 구조)
- [ ] public_html/assets/js/certPrint.js (JavaScript 로직)
  - 참조: grdList.js (hr_tbl 초기화, 이벤트 핸들러)
- [ ] public_html/sys/certPrintConfig.php (CRUD 처리)
  - 참조: grdConfig.php (INSERT/UPDATE/DELETE 쿼리 구조)
- [ ] public_html/sys/certPrintList.php (데이터 조회)
  - 참조: grdList.php (WHERE 조건, 파라미터 바인딩)
- [ ] public_html/assets/css/certPrint.css (페이지 전용 스타일)
  - 참조: grdList.css (필요 시)

## 데이터베이스 설계
- [ ] BONDANG_HR.CERTIFICATE_HISTORY 테이블 설계
  - [ ] CERT_CD (증명서 일련번호)
    - 타입: VARCHAR(20)
    - 설명: 증명서 고유 식별자
    - 참조: grdList.GRD_CD 구조
  - [ ] PSNL_CD (직원코드)
    - 타입: VARCHAR(20)
    - 설명: 직원 고유 식별자
    - 참조: grdList.PSNL_CD 구조
  - [ ] CERT_TYPE (증명서 종류)
    - 타입: VARCHAR(50)
    - 설명: 증명서 종류 (재직증명서, 경력증명서 등)
    - 참조: adjList.ADJ_NUM 구조
  - [ ] ISSUE_DT (발급일자)
    - 타입: DATE
    - 설명: 증명서 발급일
    - 참조: grdList.ADVANCE_DT 구조
  - [ ] CERT_DTL (증명서 상세 내용)
    - 타입: TEXT
    - 설명: 증명서 상세 내용
    - 참조: grdList.GRD_DTL 구조
  - [ ] REG_DT (등록일시)
    - 타입: DATETIME
    - 설명: 데이터 등록일시
    - 참조: grdList.REG_DT 구조

## 참고 사항
- grdList 페이지의 구조와 기능을 최대한 참조
- 기존 컴포넌트(header.php, footer.php) 재사용
- 기존 JavaScript 라이브러리(hr_tbl.js, modal.js) 활용
- 보안을 위해 API 키 검증 로직 추가
- CSS 파일은 기존 파일들(hr_tbl.css, modal.css, searchArea.css) 재사용 원칙
- 필요 시에만 certPrint.css 신규 생성
- 증명서 종류는 셀렉트 박스로 구현 (재직증명서, 경력증명서, 재직예정증명서 등)
- 발급일자는 dateBox 클래스로 날짜 형식 자동 하이픈 추가
- 직원검색은 psnlPopup.php 팝업 활용
